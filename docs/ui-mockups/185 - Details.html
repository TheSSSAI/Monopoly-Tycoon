Responsive Specifications

Desktop Layout

  • The tutorial overlays a simplified, static version of the main gameplay screen.
  • Instructional dialogs are positioned dynamically near the highlighted element to provide context.
  • The layout is optimized for 16:9, 16:10, and 21:9 aspect ratios.

Accessibility Features

Keyboard Navigation

In each step, focus is programmatically managed, allowing the user to activate the highlighted element with the keyboard.

Screen Reader Support

Instructional text in dialogs is accessible. Non-interactive elements are disabled to guide screen reader focus.

Focus Management

A key feature of the tutorial is managing user focus through visual highlights and disabling irrelevant controls.

Color Contrast

Implementation Notes

Embedded Css Architecture

All styles are embedded. A base layer styles the mocked game UI. An overlay layer styles the tutorial components (scrim, highlights, dialogs). A state-based approach using CSS selectors demonstrates the progression through different steps.

Screen Organization

The HTML is structured with a background layer for the game UI and a foreground layer for the tutorial overlays. Each tutorial step is a self-contained element.

Responsive Design

The background game UI uses the standard responsive layout. The tutorial overlays use absolute positioning with CSS variables to dynamically place highlights and dialogs.

Accessibility Compliance

Focus is visually guided. Non-interactive elements are disabled with `pointer-events: none` to prevent accidental clicks.

Browser Support

Modern browsers. The use of CSS variables and advanced selectors is key.

Usage Guidelines

When Accessed

When a new user wants a hands-on introduction to the game's controls and rules, accessed from the Main Menu.

Primary Actions

Following on-screen instructions to click highlighted buttons and learn game concepts.

Navigation Patterns

The flow is linear, progressing from one step to the next upon completing the required action. The user can exit at any time via a pause menu.

Success Criteria

The user successfully completes all steps and understands the basic gameplay loop.

Testing Recommendations

Screen Testing

Test the entire linear flow of the tutorial, from start to completion. Verify that UI highlights and instructional dialogs appear correctly for each step.

Responsive Testing

Verify on 16:9, 16:10, and 21:9 aspect ratios that the highlights and dialogs remain correctly positioned relative to their target elements.

Accessibility Testing

  • Confirm that at each step, only the intended interactive element is focusable via keyboard.
  • Check color contrast of tutorial text and highlights.
  • Ensure instructional text is clear and concise.

User Flow Testing

Test the premature exit flow from multiple different steps in the tutorial. Test both navigation options on the completion screen.

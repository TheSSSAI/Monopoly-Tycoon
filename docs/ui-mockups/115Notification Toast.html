Notification Toast

Provides non-intrusive, auto-dismissing feedback to the user.

Component Variations

The component has three main variants based on semantic colors: Info, Success, and Error. In a live application, these would appear in the top-center of the screen and fade in/out.

Component Variations

Variant Name: Info Toast

Used for neutral, informational messages that keep the player aware of game state changes, such as AI-to-AI trades.

AI 1 and AI 2 completed a trade.

Variant Name: Success Toast

Used to confirm that a player's action was completed successfully, such as saving the game or changing settings.

Game settings saved successfully!

Variant Name: Error Toast

Used to provide feedback on a failed, non-critical operation. Critical errors should use a modal dialog instead.

Could not save game. Please check disk space.

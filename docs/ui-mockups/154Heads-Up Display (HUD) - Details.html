Component Name

Heads-Up Display (HUD)

Component Type

organism

Description

A persistent, anchored layout that displays real-time status panels for each player in the game. It provides at-a-glance information on player names, tokens, cash, and current turn status.

Design System Alignment

Uses design system tokens for colors, typography, spacing, and shadows to create consistent player status panels. The layout is anchored, a key pattern for in-game overlays.

Responsive Specifications

Mobile Adaptations

  • N/A for this desktop-exclusive application.

Tablet Adaptations

Desktop Enhancements

  • Player panels are absolutely positioned and anchored to the four corners of the viewport.
  • This layout ensures that the central game board area remains unobstructed regardless of aspect ratio (16:9, 16:10, 21:9).
  • The HUD container uses `pointer-events: none` to allow interaction with the game world, while individual panels have `pointer-events: auto`.

Accessibility Features

Keyboard Support

The HUD is a display-only component and is not interactive.

Screen Reader Support

Information should be conveyed through game events. The panels themselves are decorative containers.

Aria Attributes

    Color Contrast

    Player name and cash text meet WCAG AA contrast ratios against the panel background. The 'Active Turn' state is indicated by both a colored shadow/border and a scale transform, not just color alone.

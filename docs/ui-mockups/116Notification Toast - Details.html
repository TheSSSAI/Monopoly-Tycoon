Description

A non-intrusive, auto-dismissing notification component used to provide timely feedback for non-critical game events, such as AI-to-AI trades or successful data operations. It appears in a consistent screen location and disappears after a short duration.

Design System Alignment

This component utilizes the design system's semantic colors for its variants (success, error, info), card styling for its container (shadows, border-radius), and secondary font family for its text content. It's an implementation of the 'Alerts' feedback pattern.

Responsive Specifications

Mobile Adaptations

  • N/A (Desktop-only game). However, component is intrinsically responsive due to use of max-width and being anchored to the viewport.

Tablet Adaptations

  • N/A

Desktop Enhancements

  • Anchored to the top-center of the screen to provide consistent, non-intrusive feedback across all supported aspect ratios (16:9, 16:10, 21:9).

Accessibility Features

Keyboard Support

Component is non-interactive and does not require keyboard support.

Screen Reader Support

Each toast element uses `role="status"` and `aria-live="polite"` to ensure its message is announced by screen readers without interrupting the user's current task.

Focus Management

No focus management is needed as the component is non-interactive.

Aria Attributes

  • role="status"
  • aria-live="polite"

Color Contrast

All text/background color combinations for info, success, and error variants are designed to meet WCAG 2.1 AA contrast ratios.

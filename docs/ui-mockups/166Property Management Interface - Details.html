Component Name

Property Management Interface

Component Type

organism

Description

A modal interface for players to view and manage their owned properties, including building, selling, mortgaging, and trading. It is presented as an overlay, pausing the main game flow to allow for focused strategic decisions.

Design System Alignment

This organism is composed of atomic components like Buttons and Cards. It heavily utilizes design system tokens for typography, color, spacing, and shadows to maintain visual consistency with the rest of the application.

Responsive Specifications

Mobile Adaptations

  • N/A (Desktop-only game)

Tablet Adaptations

  • N/A (Desktop-only game)

Desktop Enhancements

  • The modal is constrained to a max-width and max-height to remain usable on very large screens.
  • The property list is vertically scrollable to accommodate a large number of properties.
  • The property cards are laid out in a responsive grid that wraps based on available width.

Accessibility Features

Keyboard Support

The modal should trap focus. All interactive elements (close button, property cards, action buttons) should be navigable using the Tab key. Enter/Space should activate buttons.

Screen Reader Support

The modal should have an `aria-labelledby` attribute pointing to the title. Property groups and cards should be structured with headings for easy navigation. Disabled buttons should be programmatically disabled to be announced correctly.

Focus Management

Focus is moved to the modal container on open. Visible focus rings are applied to all interactive elements.

Aria Attributes

  • aria-modal="true"
  • role="dialog"
  • aria-labelledby

Color Contrast

All text and button combinations adhere to WCAG AA standards. Property color groups are identified by a colored border and a text heading.

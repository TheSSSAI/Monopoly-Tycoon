Component Name

Property Card Display

Component Type

molecular

Description

Displays the details of a Monopoly property, including its name, color group, rent structure, development costs, and current status (e.g., mortgaged, developed). It comes in a full 'deed' view and a compact 'summary' view for lists.

Design System Alignment

This component is built upon the atomic 'Card' specification. It uses the design system's typography scale, color palette for property groups and states, and spacing system for internal layout.

Responsive Specifications

Mobile Adaptations

  • Not applicable. Component is designed for desktop aspect ratios. It scales within its parent container.

Tablet Adaptations

  • Not applicable.

Desktop Enhancements

  • Hover states on summary cards provide interactive feedback.
  • Clear visual hierarchy for at-a-glance reading of property details.

Accessibility Features

Keyboard Support

The summary card can be made focusable if it's part of a list that requires keyboard navigation.

Screen Reader Support

Semantic structure (headers, lists) helps screen readers interpret the card content. Mortgaged state should be announced if the card is part of an interactive list.

Focus Management

Relies on parent organism for focus management.

Aria Attributes

Can use `aria-label` or `aria-describedby` on the card's root element in a list context to provide a full summary for screen reader users.

Color Contrast

All text meets WCAG AA contrast ratios against its immediate background. Property color banners contain only white text, ensuring high contrast.

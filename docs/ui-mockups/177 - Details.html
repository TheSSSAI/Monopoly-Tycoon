Responsive Specifications

Desktop Layout

  • HUD elements are anchored to the four corners of the screen.
  • Central action buttons are anchored to the bottom-center.
  • The main game board area occupies the central space.
  • Modals are centered on the screen with a background scrim.

Tablet Adaptations

  • N/A. Follows desktop layout principles.

Mobile Adaptations

  • N/A. This is a Windows-exclusive desktop application.

Accessibility Features

Keyboard Navigation

Logical tab order from pause menu button to main action buttons, then to modal dialogs when active.

Screen Reader Support

Not a primary requirement for this Unity game, but all interactive elements have clear text labels. Important status changes (like player turn) are indicated by non-color-based means (e.g., brightness, border) in addition to color.

Focus Management

Focus is trapped within modal dialogs when they are active. A clear focus ring is present on interactive elements.

Aria Attributes

    Color Contrast

    WCAG 2.1 AA compliance is maintained for all text in the HUD and modals.

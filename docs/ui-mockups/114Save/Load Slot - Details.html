Component Name

Component Type

Description

A card-based component that displays the state and metadata of a game save slot. It can represent empty, occupied, corrupted, or incompatible states and supports selection.

Design System Alignment

Utilizes the 'Card' base from data display specifications, along with typography and color tokens for text and semantic states (error, warning). Interaction states follow system guidelines for focus and hover.

Accessibility Features

Keyboard Support

Component is a

Screen Reader Support

Semantic button element announces its purpose. Disabled state is programmatically communicated. All metadata is readable text.

Focus Management

A clear, high-contrast focus ring is applied on :focus-visible.

Aria Attributes

  • aria-label on the button combines all info for a concise screen reader announcement, e.g., 'Save Slot 1, BoardwalkKing, Turn 45. Press to select.'
  • aria-disabled='true' is implicitly set by the disabled attribute on corrupted/incompatible slots.

Color Contrast

All text/background combinations meet WCAG AA standards. Status indicators (corrupted, incompatible) use text labels in addition to color.

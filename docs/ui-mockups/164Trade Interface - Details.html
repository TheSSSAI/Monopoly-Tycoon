Description

A comprehensive two-panel modal interface for proposing, reviewing, and responding to trade offers between the human player and an AI opponent. It handles player-initiated trades, AI-initiated offers, and counter-offers.

Design System Alignment

Utilizes the design system's color palette, typography, spacing, and button/input component specifications. The modal structure follows the established interaction pattern for turn-halting decisions.

Responsive Specifications

Mobile Adaptations

  • N/A. This component is designed for desktop aspect ratios.

Tablet Adaptations

  • N/A

Desktop Enhancements

  • The modal uses a max-width to remain legible on ultra-wide screens.
  • The two-panel layout is optimized for comparing assets side-by-side.
  • Scrollable inventory lists handle cases where players own many properties.

Accessibility Features

Keyboard Support

Full keyboard navigation for all interactive elements (buttons, inputs, asset lists) is required. Focus should be trapped within the modal.

Screen Reader Support

The modal should have an `aria-labelledby` attribute pointing to its title. All buttons have descriptive text. Asset lists should be navigable.

Focus Management

Focus is moved to the first interactive element when the modal opens and is trapped inside until closed. Visible focus rings are applied to all elements.

Aria Attributes

  • aria-modal="true"
  • role="dialog"
  • aria-labelledby

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast requirements. Mortgaged status is indicated by both color and text.

Implementation Notes

Embedded Css Architecture

All styles are embedded, using design system tokens as CSS custom properties. A BEM naming convention is used for the organism (.trade-interface), its elements (.trade-interface__header), and modifiers (.asset-item--mortgaged). Atomic component styles are also included.

Component Organization

The HTML is structured to showcase the primary use case (player-initiated trade) and then demonstrate its variants (AI-initiated response, counter-offer) in a separate section.

Responsive Design

The design is fixed-fluid, with a max-width on the modal. It will remain centered and functional across all target desktop aspect ratios (16:9, 16:10, 21:9).

Accessibility Compliance

Focus management, semantic roles, and clear labeling are the primary accessibility concerns. Disabled states are clearly communicated visually.

Browser Support

Targets modern browsers. Self-contained within the HTML file.

Usage Guidelines

When To Use

Use whenever a trade is initiated by the player, or when an AI proposes a trade to the player.

Content Guidelines

Asset names must be displayed fully. Cash values should be formatted as currency.

Placement Rules

Appears as a modal dialog, centered on the screen, with a scrim overlay that blocks interaction with the background game.

Customization Options

  • Use `.trade-interface--propose-mode` for player-initiated and counter-offer states.
  • Use `.trade-interface--response-mode` for AI-initiated offers.
  • Individual assets can have modifier classes like `.asset-item--mortgaged` or `.inventory-item--disabled`.

Testing Recommendations

Browser Testing

Test in Chrome, Firefox, Safari, Edge.

Device Testing

Verify layout and functionality on 16:9, 16:10, and 21:9 aspect ratios.

Accessibility Testing

  • Full keyboard navigation testing (Tab, Shift+Tab, Enter, Space, Arrow keys for lists).
  • Screen reader check for proper announcements of modal title and actions.
  • Color contrast validation for all text, especially mortgaged indicators.

Performance Testing

Ensure smooth opening/closing animations and no lag when adding/removing multiple items from the offer.

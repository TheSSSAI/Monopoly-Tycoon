Screen Variations

This is a static mockup. The variants below describe the different interactive states of this screen.

Player's Turn (Default)

This is the default view. The human player's HUD panel is highlighted (.hud-player-card--active). The main action buttons ("Roll Dice", "Manage Properties") are visible and enabled.

AI's Turn

An AI player's HUD panel would have the --active class. The human player's action buttons would be hidden or replaced by a disabled "AI Turn in Progress..." button (as shown by `id="game-actions-ai-turn"`).

Awaiting Decision

The modal scrim (#decision-modal) would get an .is-visible class. This blurs and darkens the background, showing a centered dialog for a critical decision, like buying a property. All background UI is non-interactive.

Paused

The pause overlay (#pause-screen) would get an .is-visible class. This covers the screen and presents menu options like "Resume", "Save", and "Settings".
